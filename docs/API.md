## 🎯 CLI Commands

### `custom-push init`

Main command to set up push notifications in your project.

```bash
npx custom-push init [options]
```

#### Options

| Option | Type | Description |
|--------|------|-------------|
| `--help` | flag | Show help message |
| `--version` | flag | Show CLI version |
| `--verbose` | flag | Enable verbose logging |

#### Process Flow

1. **Project Detection** - Analyzes package.json and project structure
2. **Version Validation** - Checks Firebase/React compatibility  
3. **Interactive Prompts** - Asks for undetectable configuration
4. **File Generation** - Creates service worker and helper files
5. **Backend Setup** - Scaffolds backend code if detected

#### Exit Codes

| Code | Meaning |
|------|---------|
| 0 | Success |
| 1 | Error (missing package.json, invalid input, etc.) |
| 2 | User cancelled |

## 🎨 Frontend API

### `usePush()`

Main React hook for initializing push notifications.

```typescript
import { usePush } from './push/pushHelper'

interface UsePushOptions {
  onMessage?: (payload: MessagePayload) => void
  onReady?: (token: string) => void
  onPermissionDenied?: () => void
  onError?: (error: Error) => void
}

function usePush(options?: UsePushOptions): void
```

#### Usage

```typescript
import { usePush } from './push/pushHelper'

function App() {
  usePush({
    onMessage: (payload) => {
      console.log('Foreground message:', payload)
      // Show toast notification
      toast(`${payload.notification?.title}: ${payload.notification?.body}`)
    },
    onReady: (token) => {
      console.log('FCM token:', token)
      // Send token to your backend
      registerToken(token)
    },
    onPermissionDenied: () => {
      console.warn('Notification permission denied')
      // Show UI to request permission
    },
    onError: (error) => {
      console.error('Push error:', error)
      // Show error message to user
    }
  })
  
  return <YourApp />
}
```

## 🔧 Backend API

Generated Express routes for token management.

```typescript

```

#### POST `/push/unregister`

Removes a device token.

```typescript
interface UnregisterRequest {
  token: string
}

  success: boolean
  error?: string
}
```

### NestJS Module

Generated NestJS module with service and controller.

#### PushService

```typescript
@Injectable()
export class PushService {
  // Register device token
  async registerToken(userId: string, token: string): Promise<void>
  
  // Unregister device token  
  async unregisterToken(token: string): Promise<void>
  
  // Send push notification
  async sendNotification(params: SendNotificationParams): Promise<string>
}
```

#### PushController

```typescript
@Controller('push')
export class PushController {
  @Post('register')
  async register(@Body() body: RegisterTokenDto): Promise<RegisterResponse>
  
  @Post('unregister') 
  async unregister(@Body() body: UnregisterTokenDto): Promise<UnregisterResponse>
}
```

## 🎪 Events

### Frontend Events

The push system emits several events during its lifecycle:

#### `push:ready`
 Fired when push notifications are ready and token is available.

```typescript
usePush({
  onReady: (token) => {
    console.log('Push ready, token:', token)
    // Send token to backend
  }
})
```

#### `push:message`
Received when a push notification arrives while app is in foreground.

```typescript
usePush({
  onMessage: (payload) => {
    console.log('Push message:', payload)
    // Handle foreground notification
  }
})
```

#### `push:permission_denied`
Fired when user denies notification permission.

```typescript
usePush({
  onPermissionDenied: () => {
    console.log('Permission denied')
    // Show custom permission UI
  }
})
```

#### `push:error`
Fired when any error occurs.

```typescript
usePush({
  onError: (error) => {
    console.error('Push error:', error)
    // Handle error gracefully
  }
})
```

### Service Worker Events

The generated service worker handles these events automatically:

#### `notificationclick`
Handles notification click events and navigates to routes.

#### `push`
Handles background push messages.

#### `notificationclose`
Handles notification close events.

## 🚨 Error Handling

### Common Errors

| Error | Cause | Solution |
|-------|-------|----------|
| `No package.json found` | Not in project root | Run from project directory |
| `Invalid Firebase config` | Missing required fields | Check Firebase Console settings |
| `Permission denied` | Notification permission blocked | Ask user to enable in browser settings |
| `Token registration failed` | Backend endpoint error | Check backend implementation |
| `Service worker error` | Invalid service worker | Check console for errors |
